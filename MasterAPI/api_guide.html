Master API Documentation

Reference guide for the Fonarev 360 Master Data Hierarchy API

Base URL

All endpoints are prefixed with:

http://localhost:3005/api/v1/master-data

Hierarchy Endpoints

GET /departments

Retrieves a list of all distinct departments.

Response Example:

[
  "ACCES A LA JUSTICE",
  "AUDIT INTERNE",
  "COMMUNICATION",
  "DIRECTION FINANCIÈRE"
]
GET /activities

Retrieves activities for a specific department.

Query Parameters:

  • department Required - The name of the department (e.g., "COMMUNICATION")

Example Request:

GET /activities?department=COMMUNICATION

Response Example:

[
  "Relations publiques",
  "Participation aux conférences",
  "Campagnes digitales"
]
GET /sub-activities

Retrieves sub-activities for a specific activity within a department.

Query Parameters:

  • department Required
  • activity Required

Example Request:

GET /sub-activities?department=...&activity=...
GET /tasks

Retrieves list of tasks (budget lines) including codes.

Query Parameters:

  • department Required
  • activity Required
  • sub_activity Required

Response Example:

[
  {
    "id": 101,
    "name": "Achat fournitures",
    "code": "COM.01.01",
    "code_tache": "01"
  },
  ...
]

Usage Example (JavaScript)

// Function to fetch departments
async function getDepartments() {
    try {
        const response = await fetch('http://localhost:3005/api/v1/master-data/departments');
        const data = await response.json();
        console.log(data); // ["Dept A", "Dept B"...]
    } catch (error) {
        console.error('Error fetching departments:', error);
    }
}

getDepartments();
